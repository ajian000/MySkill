# 战棋地图生成工具

用于开发类火焰纹章（Fire Emblem）风格战棋游戏的地图生成工具。基于正方形网格，提供完整的地图设计、编辑和导出能力。

## 何时使用

- 用户要求"生成一个战棋地图"或"制作火纹风格地图"
- 用户需要设计战棋游戏关卡地图
- 用户需要随机生成战术地图
- 用户需要将地图数据导出为游戏可读的格式

## 使用方式

### 方式一：交互式配置生成（推荐）

通过问答方式收集需求，然后调用脚本生成地图。

**必备参数**：
| 参数 | 说明 | 默认值 |
|------|------|--------|
| 地图尺寸 | 行数 × 列数（如 20×25） | 15×20 |
| 地形风格 | 平原/森林/山地/混合 | 混合 |

**可选参数**：
| 参数 | 说明 | 默认值 |
|------|------|--------|
| 水域比例 | 0~1 | 0.1 |
| 森林比例 | 0~1 | 0.15 |
| 山地比例 | 0~1 | 0.1 |
| 道路密度 | 0~1 | 0.3 |
| 城堡/村庄数量 | 整数 | 2 |
| 难度倾向 | 简单/中等/困难 | 中等 |
| 胜利条件 | 压制/全灭/防守/护送 | 压制 |
| 进攻路线图 | 是否需要生成进攻路线参考图 | 否 |

**进攻路线参考图说明**：
- 生成后会在地图文件名后追加 `_路线图.png` 的额外图片
- 图片上用**绿色区域**标注玩家出生位置，**红色区域**标注敌方布防位置
- 用**橙色路径**绘制 BFS 算法计算出的推荐进攻路线
- 帮助设计师直观评估关卡布局和推进节奏
- 需要询问用户是否需要此功能

对话完成后，调用脚本生成地图。**在输出地图前，必须询问用户是否需要生成进攻路线参考图**。

### 方式二：直接指定参数生成

用户直接提供完整参数后，调用脚本生成。

**示例命令**：
```bash
python .trae/skills/战棋地图生成工具/scripts/map_generator.py \
  --rows 20 --cols 25 \
  --water 0.1 --forest 0.15 --mountain 0.1 \
  --villages 3 --roads 0.3 \
  --difficulty medium --objective seize

生成带进攻路线参考图：
```bash
python .trae/skills/战棋地图生成工具/scripts/map_generator.py \
  --rows 20 --cols 25 --random --route \
  --output outputs/关卡地图
```
```

### 方式三：手动编辑已有地图

用户提供已有的地图 JSON 文件路径，基于该文件进行编辑修改。

```bash
python .trae/skills/战棋地图生成工具/scripts/map_generator.py \
  --edit 已有地图.json \
  --terrain 3,5=mountain \
  --player-start 0,0 \
  --enemy-zone 18,20
```

## 角色参考位置标注

地图支持标注以下角色参考位置：

### 我方单位参考位置
- 用绿色标记显示我方推荐出场位置
- 支持指定单位类型（如：领主、剑士、法师、治疗师等）
- 示例标注格式：第 0 行第 0~4 列为玩家起始区域

#### PNG 图片中的视觉标记
- **玩家起始区域** — 格子带绿色边框标注
- **我方单位图标** — 红色边框 + 蓝色填充圆形，旁边显示职业名称（如"领主"、"剑士"）

### 敌方布防参考位置
- 用红色标记显示敌方推荐布防区域
- 支持指定单位类型（如：步兵、骑兵、飞行单位、Boss 等）
- 示例标注格式：第 18~20 行第 20~24 列为敌方区域

#### PNG 图片中的视觉标记
- **敌方布防区域** — 格子带红色边框标注
- **敌方集群图标** — 黑色边框 + 红色填充圆形，标注 `x3-5` 表示该位置代表 3~5 个敌方单位的集群
- 一个敌方标记不代表仅该格子有敌人，而是指该区域有一支 3~5 单位的敌方部队

### 中立/NPC 位置
- 用蓝色标记显示 NPC 或中立单位位置

## 进攻路线参考图

生成一张独立的标注图片，用于辅助关卡设计分析：

### 图片内容
- **绿色区域** — 标注玩家出生位置（我方起始区），中心处有"玩家出生"标签
- **红色区域** — 标注敌方布防位置，中心处有"敌方布防"标签
- **橙色路径** — 使用 BFS 算法从玩家区到敌方区计算的最短可行路径，路径节点和连线清晰可见
- **半透明地形底图** — 保留地图地形作为背景参考
- **单位图标** — 同上，红色边框蓝色填充圆形表示我方单位，黑色边框红色填充圆形（带 `x3-5`）表示敌方集群

### 设计用途
- 直观检查玩家出生点到目标点的距离是否合理
- 判断路线是否会经过特定地形（如桥头、森林等战术要点）
- 评估关卡推进节奏，避免路线过长或过短
- 辅助决定伏兵位置、对话触发点等关卡事件放置

## 输出文件说明

生成的地图包含以下文件：

| 文件 | 格式 | 说明 |
|------|------|------|
| `地图名称.json` | JSON | 完整地图数据，可直接被游戏引擎读取 |
| `地图名称.txt` | ASCII | 文本格式地图，适合快速预览 |
| `地图名称.png` | 图片 | 可视化地图截图（需安装 PIL） |
| `地图名称_路线图.png` | 图片 | 进攻路线参考图（需安装 PIL，仅 `--route` 时生成） |

## 随机地图生成算法

### 生成流程
1. **初始化** - 创建全平原的空白网格
2. **地形分区** - 使用柏林噪声或随机分块算法分布地形
3. **水域生成** - 生成河流和湖泊，确保不阻断关键路径
4. **道路生成** - 使用 A* 算法连接关键地点
5. **关键点放置** - 放置城堡、村庄等地标
6. **边界处理** - 地图边界自动添加悬崖/山脉
7. **玩家区域** - 在地图底部划分我方起始区
8. **敌方区域** - 在地图顶部划分敌方布防区
9. **可通行验证** - 确保地图所有关键位置间存在可通行路径

### 连通性保证
- 自动检测并确保玩家起始区到目标点之间存在至少一条可行路径
- 不可通行地形（河流若无桥梁、山脉、城墙）不会阻断关键路径

## 使用示例

### 示例 1：生成标准地图

用户说："生成一个 20×25 的平原城堡攻防战地图"

AI 的操作流程：
1. 与用户确认具体参数
2. 询问用户是否需要生成**进攻路线参考图**
3. 调用脚本生成地图
4. 展示生成的 JSON 数据、ASCII 预览和 PNG 图片
5. 如生成了路线图，也一并展示并说明"绿色=玩家出生、红色=敌方布防、橙色=推荐进攻路径"
6. 询问用户是否需要调整

### 示例 2：随机生成

用户说："随机生成一张适合第 5 章的中等难度地图"

AI 的操作流程：
1. 根据"第 5 章"推断合适的地图尺寸和敌方强度
2. 询问用户是否需要生成**进攻路线参考图**
3. 调用脚本随机生成
4. 展示结果并让用户反馈修改意见

### 示例 3：手动精细化编辑

用户说："把 3 行 5 列改成森林，在第 10 行 12 列放一个要塞"

AI 的操作流程：
1. 读取当前地图 JSON 数据
2. 询问用户是否需要生成**进攻路线参考图**
3. 使用编辑功能修改指定格子的地形
4. 重新生成输出文件（如需路线图则加 `--route`）
5. 展示修改结果

## 依赖安装

如果需要生成 PNG 可视化图片，需要安装 Pillow：

```bash
pip install Pillow
```

核心功能（JSON + ASCII 输出）不依赖任何第三方库。
